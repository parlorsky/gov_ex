# Запросы к базам данных с использованием нескольких таблиц. Объединение и пересечение отношений. Соединения отношений — основные виды соединений, их характеристики.
### JOIN-ы
- **INNER JOIN ... ON cond** — только пары, удовлетворяющие условию.
- **LEFT JOIN** — все строки левой + совпадения справа (NULL, если нет).
- **RIGHT JOIN** — зеркально (редко используется).
- **FULL OUTER JOIN** — все строки обеих таблиц, несовпадения с NULL. = INNER + одиночки с обеих сторон.
- **CROSS JOIN** — декартово произведение $n \times m$, без `ON`. Случайно возникает, если забыли условие. Намеренно — для генерации всех комбинаций.
- **SELF JOIN** — таблица с самой собой по разным алиасам. Иерархии (`manager_id`), LAG/LEAD без оконных функций.
- **NATURAL JOIN** — по одноимённым столбцам автоматически. Опасен: добавление столбца меняет семантику. Лучше явный `ON`.

**Эквисоединение** — условие через `=`. **Тета-соединение** — через любой предикат (`<`, `BETWEEN`).

### Множественные операции
- **UNION** — объединение с дедупликацией.
- **UNION ALL** — без дедупликации (быстрее).
- **INTERSECT** — пересечение.
- **EXCEPT** (или `MINUS` в Oracle) — разность.

**Требования:** одинаковое число столбцов, совместимые типы. Имена — из первого запроса.

### Подзапросы
- **Скалярный** — возвращает одно значение, используется в `WHERE`/`SELECT`.
- **В FROM** — производная таблица: `FROM (SELECT...) AS t`.
- **IN / NOT IN** — список значений.
- **EXISTS / NOT EXISTS** — проверка наличия. Часто эффективнее IN на больших данных, корректно работает с NULL.
- **Коррелированный** — ссылается на колонки внешнего запроса, выполняется для каждой строки внешнего. Медленнее, но позволяет условия "относительно текущей строки".

### CTE (`WITH`)

Именованный временный результат, можно переиспользовать в запросе. Чище подзапросов, рекурсивные CTE — для иерархий и графов через `WITH RECURSIVE`.

### Когда что
- Нужны столбцы из обеих таблиц → **JOIN**.
- Проверить наличие → **EXISTS**.
- Список фиксированных значений → **IN (...)**.
- Объединение результатов разных запросов → **UNION**.
- Структурный повторно используемый шаг → **CTE**.

### Подводные камни
- **CROSS JOIN случайно** — забыли `ON` или `WHERE`, получили $n \times m$ строк.
- **LEFT JOIN + WHERE на правой таблице** — превращает LEFT в INNER, потому что фильтр выкидывает NULL-строки. Решение: условие в `ON`, не в `WHERE`.
- **NOT IN + NULL в подзапросе** — возвращает пусто из-за тернарной логики. Всегда использовать `NOT EXISTS`.
- **UNION медленный** из-за сортировки и дедупликации. Если дубликаты невозможны — `UNION ALL`.
- **NATURAL JOIN** хрупок к изменениям схемы.

### Главное на экзамене
1. **4 основных JOIN-а** + CROSS + SELF — назначение и разница.
2. **UNION/INTERSECT/EXCEPT** + требование к структуре.
3. **EXISTS vs IN** — главное практическое различие: поведение с NULL.
4. **Два главных подводных камня:** LEFT JOIN + WHERE, NOT IN + NULL.
5. **CTE** — что это и зачем.
