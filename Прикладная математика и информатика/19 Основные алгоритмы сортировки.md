# Основные алгоритмы сортировки. Их сложность, достоинства и недостатки. Алгоритм быстрого поиска в отсортированном массиве. Сложность поиска в отсортированном и не отсортированном массиве.

## TL;DR

- **Основные алгоритмы сортировки:** простые алгоритмы работают за $O(n^2)$, эффективные сравнительные - за $O(n\log n)$, а <a href="#линейные-сортировки">линейные сортировки</a> используют структуру ключей и могут дать $O(n+k)$.
- **Их сложность:** оценивать нужно лучшее, среднее и худшее время, память, стабильность и режим in-place; сводка дана в <a href="#сравнительная-таблица-сортировок">таблице сортировок</a>.
- **Достоинства и недостатки:** <a href="#простые-сортировки">простые сортировки</a> удобны для обучения и малых массивов, quicksort быстр на практике, mergesort стабилен, heapsort даёт гарантию $O(n\log n)$.
- **Алгоритм быстрого поиска в отсортированном массиве:** <a href="#бинарный-поиск">бинарный поиск</a> на каждом шаге отбрасывает половину диапазона и работает за $O(\log n)$.
- **Сложность поиска в отсортированном и не отсортированном массиве:** в отсортированном массиве бинарный поиск даёт $O(\log n)$, а без сортировки и дополнительной структуры худший случай остаётся $O(n)$.

<a id="простые-сортировки"></a>

### Простые сортировки

**Пузырьковая сортировка** последовательно сравнивает соседние элементы и меняет их местами, если порядок неправильный. После каждого прохода один из максимальных элементов "всплывает" в конец.

```python
def bubble_sort(a):
    a = a[:]
    n = len(a)
    for i in range(n):
        swapped = False
        for j in range(0, n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
                swapped = True
        if not swapped:
            break
    return a
```

**Сортировка вставками** поддерживает отсортированный префикс и вставляет очередной элемент в нужное место. Поэтому она эффективна на почти отсортированных данных и часто применяется внутри гибридных алгоритмов для маленьких подмассивов.

**Сортировка выбором** на каждом шаге ищет минимальный элемент в неотсортированной части и ставит его на текущую позицию. Она проста, но всегда делает $\Theta(n^2)$ сравнений.

### Эффективные сравнительные сортировки

**Быстрая сортировка** выбирает опорный элемент `pivot`, разделяет массив на элементы меньше и больше pivot, затем рекурсивно сортирует части. Средняя сложность $O(n\log n)$ получается при сбалансированных разбиениях. Худший случай $O(n^2)$ возникает, если pivot постоянно оказывается минимумом или максимумом. На практике используют случайный pivot, median-of-three или introsort.

**Сортировка слиянием** делит массив пополам, сортирует половины и сливает два отсортированных массива. Она стабильна и всегда работает за $O(n\log n)$, но обычно требует $O(n)$ дополнительной памяти.

**Пирамидальная сортировка** строит двоичную кучу. Сначала из массива строится max-heap за $O(n)$, затем максимум многократно переносится в конец, а куча восстанавливается. Время всегда $O(n\log n)$, память $O(1)$, но стабильности нет.

**Timsort** — гибрид сортировки слиянием и вставками. Он находит уже отсортированные фрагменты (`runs`), расширяет короткие фрагменты вставками и сливает их по правилам баланса. Это стандартная сортировка Python для `list.sort()` и `sorted()`.

<a id="линейные-сортировки"></a>

### Линейные сортировки

Сравнительные сортировки не могут гарантированно быть быстрее $\Omega(n\log n)$, потому что им нужно различить $n!$ возможных порядков элементов. Линейные сортировки обходят это ограничение, используя структуру ключей.

**Counting sort** считает частоты целых ключей из ограниченного диапазона. Подходит, когда $k$ не намного больше $n$.

**Radix sort** сортирует числа или строки по разрядам, обычно используя стабильную counting sort как внутренний шаг.

**Bucket sort** раскладывает элементы по корзинам, сортирует каждую корзину и объединяет результат. Хороша при равномерном распределении.

<a id="бинарный-поиск"></a>

### Бинарный поиск

Бинарный поиск применяется к отсортированному массиву или к монотонному предикату. Идея: на каждом шаге отбрасывать половину области поиска.

Для массива:

```python
def binary_search(a, x):
    left, right = 0, len(a)

    while left < right:
        mid = (left + right) // 2
        if a[mid] < x:
            left = mid + 1
        else:
            right = mid

    if left < len(a) and a[left] == x:
        return left
    return -1
```

Такой вариант ищет первое вхождение `x`. Инвариант: ответ, если он существует, находится в полуинтервале $[left, right)$.

Сложность:

- в отсортированном массиве: $O(\log n)$ по времени и $O(1)$ по памяти;
- в неотсортированном массиве без дополнительной структуры: $O(n)$, потому что в худшем случае нужно проверить каждый элемент;
- если сначала отсортировать массив ради одного поиска, цена будет $O(n\log n)$ на сортировку плюс $O(\log n)$ на поиск, что хуже линейного поиска;
- при большом числе запросов сортировка может окупиться: один раз $O(n\log n)$, затем каждый запрос $O(\log n)$.

Бинарный поиск по ответу используется, когда есть монотонный предикат:
$$
F, F, F, T, T, T.
$$

Например, можно искать минимальную грузоподъёмность корабля, при которой все грузы перевозятся за заданное число дней: если грузоподъёмность $X$ достаточна, то любая большая грузоподъёмность тоже достаточна.
