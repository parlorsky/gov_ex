# Словари в Python. Назначение, особенности, создание, изменение. Основные операции и методы. Способы итерации по словарям.

## TL;DR
`dict` — встроенный отображающий тип Python, реализован как **хеш-таблица** с открытой адресацией. Хранит пары `ключ-значение`; ключи должны быть **хешируемыми** (неизменяемыми). Доступ, вставка, удаление — $O(1)$ в среднем. Создание: `{}`, `dict(...)`, dict comprehension. Основные методы: `get`, `setdefault`, `update`, `pop`, `keys/values/items`. С Python 3.7+ сохраняется порядок вставки. Итерация — по ключам по умолчанию.

## Развёрнуто

### Назначение и особенности
- Ассоциативный массив (отображение): для каждого ключа хранится связанное значение.
- Реализация — хеш-таблица с открытой адресацией.
- **Ключи**: должны быть хешируемыми (`__hash__`) и иметь корректный `__eq__`. Стандартно — все неизменяемые встроенные типы: `int, float, str, tuple` (если все элементы хешируемы), `frozenset`, `bool`, `None`. Списки, словари, множества — нехешируемы.
- **Значения** — любые объекты.
- **Порядок** вставки сохраняется с CPython 3.7+ (часть спецификации языка).
- Сложность: средне $O(1)$ на доступ, вставку, удаление; худшее $O(n)$ (резкие коллизии или плохой хеш).

### Создание
```python
d1 = {}
d2 = {'a': 1, 'b': 2}
d3 = dict(a=1, b=2)
d4 = dict([('a', 1), ('b', 2)])
d5 = dict(zip(['a', 'b'], [1, 2]))
d6 = dict.fromkeys(['a','b','c'], 0)   # {'a':0,'b':0,'c':0}
d7 = {k: k**2 for k in range(5)}        # dict comprehension
```

### Доступ и установка
```python
d['a']             # KeyError, если нет
d.get('a')         # None, если нет
d.get('a', 0)      # default 0
d['a'] = 10        # установка/обновление
'a' in d           # проверка наличия (O(1) в среднем)
```

### Удаление
```python
del d['a']         # KeyError, если нет
v = d.pop('a')     # удалить и вернуть значение
v = d.pop('a', None)   # с default
k, v = d.popitem() # удалить последнюю пару (LIFO с 3.7+)
d.clear()
```

### Основные методы
- `d.keys()`, `d.values()`, `d.items()` — **view** объекты, отражают текущее состояние словаря.
- `d.update(other)` — слияние; `other` может быть словарём или итерируемым из пар. Существующие ключи перезаписываются.
- `d.setdefault(key, default)` — вернуть `d[key]`, если есть; иначе установить `default` и вернуть.
- `d.copy()` — поверхностная копия. Для глубокой — `copy.deepcopy(d)`.
- Слияние словарей: `d1 | d2` (3.9+), `{**d1, **d2}` (3.5+).

### Итерация
```python
for k in d:                     # по ключам
    ...
for k in d.keys():              # эквивалентно
    ...
for v in d.values():
    ...
for k, v in d.items():
    ...
```
Изменение `d` во время итерации — `RuntimeError`. Если нужно — итерировать по `list(d.keys())`.

### Реализация (CPython)
- Открытая адресация с **псевдослучайным зондированием** (perturbation) и небольшим load factor (~2/3).
- При превышении load factor — ресайзинг (как правило, удвоение размера). Все ключи перехешируются.
- С 3.6 — **компактные словари**: отдельный массив записей в порядке вставки + массив индексов, что экономит память и обеспечивает упорядоченность.
- `__hash__` для строк рандомизирован между запусками (защита от DoS-атак на хеш).

### Производные структуры (`collections`)
- `defaultdict(default_factory)` — при отсутствии ключа автоматически создаёт значение `default_factory()`. Удобно для аккумуляции в группы и счётчиков.
- `Counter(iterable)` — хеш-таблица «элемент → количество». Методы `most_common(n)`, арифметика `+, -, &, |`.
- `OrderedDict` — упорядоченный словарь; до 3.7 — единственный способ; теперь — для случаев, когда нужны методы `move_to_end`, явное равенство по порядку.
- `ChainMap` — цепочка словарей, поиск идёт от первого к последнему.

### Пример: подсчёт частот
```python
from collections import Counter
words = ['a', 'b', 'a', 'c', 'b', 'a']
cnt = Counter(words)            # Counter({'a':3,'b':2,'c':1})
cnt.most_common(2)              # [('a',3), ('b',2)]
```

### Подводные камни
- Изменяемые ключи запрещены — `{[1,2]: "x"}` бросает `TypeError: unhashable type: 'list'`.
- `dict.fromkeys(keys, [])` — все значения ссылаются **на один и тот же** список; чтобы получить отдельные, используйте comprehension.
- View-объекты `keys/values/items` отражают текущее состояние и не «замораживаются».
- Нельзя сравнивать словари по `<`/`>` (только `==/!=` поэлементно).
