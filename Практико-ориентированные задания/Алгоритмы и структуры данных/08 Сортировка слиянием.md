# Задача 8. Сортировка слиянием.

## Условие
Реализуйте алгоритм сортировки слиянием на языке программирования по вашему выбору. Покажите его применимость на массивах разной длины и оцените временную сложность для разных случаев.

## Решение

### Программа минимум

```python
def merge_sort(numbers):
    if len(numbers) <= 1:
        return list(numbers)

    middle = len(numbers) // 2
    left_part = numbers[:middle]
    right_part = numbers[middle:]

    sorted_left = merge_sort(left_part)
    sorted_right = merge_sort(right_part)

    result = []
    left_index = 0
    right_index = 0

    while left_index < len(sorted_left) and right_index < len(sorted_right):
        if sorted_left[left_index] <= sorted_right[right_index]:
            result.append(sorted_left[left_index])
            left_index += 1
        else:
            result.append(sorted_right[right_index])
            right_index += 1

    while left_index < len(sorted_left):
        result.append(sorted_left[left_index])
        left_index += 1

    while right_index < len(sorted_right):
        result.append(sorted_right[right_index])
        right_index += 1

    return result
```

### Полное решение

### Идея
Классический «разделяй и властвуй»:
1. Делим массив пополам;
2. Рекурсивно сортируем половины;
3. Сливаем две отсортированные части за один линейный проход.

Соотношение Мастер-теоремы: $T(n) = 2T(n/2) + \Theta(n) \;\Rightarrow\; T(n) = \Theta(n \log n)$.

### Код

```python
def merge_sort(a):
    a = list(a)
    if len(a) <= 1:
        return a
    m = len(a) // 2
    L, R = merge_sort(a[:m]), merge_sort(a[m:])
    out, i, j = [], 0, 0
    while i < len(L) and j < len(R):
        if L[i] <= R[j]: out.append(L[i]); i += 1
        else:            out.append(R[j]); j += 1
    out.extend(L[i:]); out.extend(R[j:])
    return out
```

### Прогон

| $n$     | случайный |
|---------|-----------|
| 1 000   | 0,87 мс   |
| 10 000  | 11,2 мс   |
| 100 000 | 149 мс    |

Время растёт как $\sim n \log n$: при росте $n$ в 10 раз — рост времени примерно в 13 раз ($\log_{10}$ дополнительно).

### Сложности
- **Время — $\Theta(n \log n)$** во всех случаях (best/avg/worst), независимо от исходного порядка.
- **Память — $\Theta(n)$** под буферы слияния (в этом отличие от quick sort с in-place разбиением).
- **Стабильна** (в `merge` используется `<=`, поэтому одинаковые ключи из левой части идут раньше).

### Замечания
- В отличие от insertion sort и quick sort, гарантирует $O(n \log n)$ в худшем случае — поэтому востребован при необходимости предсказуемого времени.
- Базис рекурсии можно поднять до 16–32 элементов и переключаться на insertion sort — даёт ускорение в практике (пример — Timsort).
- Естественно параллелится: левую и правую половины можно сортировать независимо.
