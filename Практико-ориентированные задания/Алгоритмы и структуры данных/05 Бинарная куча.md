# Задача 5. Бинарная куча.

## Условие
Реализуйте структуру данных «Бинарная куча» на языке программирования по вашему выбору. У бинарной кучи поиска есть три свойства:
- значения в вершине не меньше, чем значения её потомков;
- глубина всех листьев отличается не более, чем на один слой;
- последний слой заполняется слева направо без «дырок».

Необходимо реализовать создание пустого дерева, добавление элемента с автоматической балансировкой дерева, удаление элемента, поиск элемента по значению. Желательно использовать ООП.

## Решение

### Программа минимум

```python
class MaxHeap:
    def __init__(self):
        self.items = []

    def push(self, value):
        self.items.append(value)
        index = len(self.items) - 1
        self.move_up(index)

    def move_up(self, index):
        while index > 0:
            parent_index = (index - 1) // 2
            if self.items[parent_index] >= self.items[index]:
                break

            temp = self.items[parent_index]
            self.items[parent_index] = self.items[index]
            self.items[index] = temp
            index = parent_index

    def find(self, value):
        for index in range(len(self.items)):
            if self.items[index] == value:
                return index
        return -1

    def pop(self):
        if len(self.items) == 0:
            return None

        max_value = self.items[0]
        last_value = self.items.pop()
        if len(self.items) == 0:
            return max_value

        self.items[0] = last_value
        self.move_down(0)
        return max_value

    def move_down(self, index):
        while True:
            left_index = 2 * index + 1
            right_index = 2 * index + 2
            biggest_index = index

            if left_index < len(self.items):
                if self.items[left_index] > self.items[biggest_index]:
                    biggest_index = left_index

            if right_index < len(self.items):
                if self.items[right_index] > self.items[biggest_index]:
                    biggest_index = right_index

            if biggest_index == index:
                break

            temp = self.items[index]
            self.items[index] = self.items[biggest_index]
            self.items[biggest_index] = temp
            index = biggest_index

    def remove(self, value):
        index = self.find(value)
        if index == -1:
            return False

        last_value = self.items.pop()
        if index < len(self.items):
            self.items[index] = last_value
            self.move_up(index)
            self.move_down(index)
        return True
```

### Полное решение

### Идея
Свойства из условия задают **полное двоичное дерево** с heap-инвариантом: значение в вершине $\ge$ значений потомков (max-куча). Полнота дерева позволяет хранить его в массиве: для узла с индексом $i$ дети — $2i+1$ и $2i+2$, родитель — $(i-1)/2$. Балансировка — это `sift_up` после вставки и `sift_down` после удаления корня.

### Код

```python
class MaxHeap:
    def __init__(self):
        self._h = []

    def __len__(self): return len(self._h)

    def push(self, v):
        self._h.append(v)
        self._sift_up(len(self._h) - 1)

    def pop(self):
        if not self._h: raise IndexError
        top = self._h[0]
        last = self._h.pop()
        if self._h:
            self._h[0] = last
            self._sift_down(0)
        return top

    def peek(self):
        if not self._h: raise IndexError
        return self._h[0]

    def search(self, v):
        for i, x in enumerate(self._h):
            if x == v: return i
        return -1

    def remove(self, v):
        i = self.search(v)
        if i == -1:
            return False
        last = self._h.pop()
        if i < len(self._h):
            self._h[i] = last
            self._sift_up(i)
            self._sift_down(i)
        return True

    def _sift_up(self, i):
        while i > 0:
            p = (i - 1) // 2
            if self._h[p] < self._h[i]:
                self._h[p], self._h[i] = self._h[i], self._h[p]
                i = p
            else: break

    def _sift_down(self, i):
        n = len(self._h)
        while True:
            l, r, best = 2*i+1, 2*i+2, i
            if l < n and self._h[l] > self._h[best]: best = l
            if r < n and self._h[r] > self._h[best]: best = r
            if best == i: break
            self._h[i], self._h[best] = self._h[best], self._h[i]
            i = best
```

### Прогон
```
push: 5,1,9,3,7,2,8 → MaxHeap[9, 7, 8, 1, 3, 2, 5]
search(7) = 1
remove(7) → True
последовательные pop: [9, 8, 7, 5, 3, 2, 1]   ← сортировка по убыванию
```

### Сложности
- `push`, `pop` — $O(\log n)$ (одно поднятие/опускание по высоте дерева).
- `remove(value)` — $O(n + \log n)$: сначала линейный поиск значения, затем восстановление кучи.
- `peek`, `len` — $O(1)$.
- `search` (по значению, не по приоритету) — $O(n)$. Это и есть «слабое место» кучи: эффективна не для произвольного поиска, а для извлечения экстремума.
- Построение из массива методом `heapify` — $O(n)$ (классический результат).

### Замечания
- Для min-кучи знаки сравнения инвертируются (или хранят `-v`).
- Свойство «последний слой заполняется слева направо» автоматически выполняется тем, что новый элемент добавляется в конец массива.
