# Типы задач машинного обучения: классификация, регрессия, кластеризация. Обучение с учителем и без учителя. Машинное обучение как оптимизационная задача.

## TL;DR
- **Типы задач машинного обучения:** основные типы разделяются по наличию меток: [обучение с учителем](<./08%20%D0%A2%D0%B8%D0%BF%D1%8B%20%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%20%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE%20%D0%BE%D0%B1%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D1%8F%20%D0%BA%D0%BB%D0%B0%D1%81%D1%81%D0%B8%D1%84%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D1%8F%2C%20%D1%80%D0%B5%D0%B3%D1%80%D0%B5%D1%81%D1%81%D0%B8%D1%8F%2C%20%D0%BA%D0%BB%D0%B0%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F.md#обучение-с-учителем>) работает с парами $(x,y)$, а [обучение без учителя](<./08%20%D0%A2%D0%B8%D0%BF%D1%8B%20%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%20%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE%20%D0%BE%D0%B1%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D1%8F%20%D0%BA%D0%BB%D0%B0%D1%81%D1%81%D0%B8%D1%84%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D1%8F%2C%20%D1%80%D0%B5%D0%B3%D1%80%D0%B5%D1%81%D1%81%D0%B8%D1%8F%2C%20%D0%BA%D0%BB%D0%B0%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F.md#обучение-без-учителя>) — только с объектами $x$.
- **Классификация:** в supervised-постановке предсказывает дискретную метку $y\in\{1,\dots,K\}$.
- **Регрессия:** в supervised-постановке предсказывает непрерывное значение $y\in\mathbb R$ и обычно оптимизирует MSE, MAE или похожий лосс.
- **Кластеризация:** в unsupervised-постановке разбивает объекты на группы похожих объектов без заранее заданных меток.
- **Обучение с учителем и без учителя:** кроме них есть [RL, semi-supervised и self-supervised](<./08%20%D0%A2%D0%B8%D0%BF%D1%8B%20%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%20%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE%20%D0%BE%D0%B1%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D1%8F%20%D0%BA%D0%BB%D0%B0%D1%81%D1%81%D0%B8%D1%84%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D1%8F%2C%20%D1%80%D0%B5%D0%B3%D1%80%D0%B5%D1%81%D1%81%D0%B8%D1%8F%2C%20%D0%BA%D0%BB%D0%B0%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F.md#другие-парадигмы>) подходы, но базовая граница проходит по наличию разметки.
- **Машинное обучение как оптимизационная задача:** [обучение](<./08%20%D0%A2%D0%B8%D0%BF%D1%8B%20%D0%B7%D0%B0%D0%B4%D0%B0%D1%87%20%D0%BC%D0%B0%D1%88%D0%B8%D0%BD%D0%BD%D0%BE%D0%B3%D0%BE%20%D0%BE%D0%B1%D1%83%D1%87%D0%B5%D0%BD%D0%B8%D1%8F%20%D0%BA%D0%BB%D0%B0%D1%81%D1%81%D0%B8%D1%84%D0%B8%D0%BA%D0%B0%D1%86%D0%B8%D1%8F%2C%20%D1%80%D0%B5%D0%B3%D1%80%D0%B5%D1%81%D1%81%D0%B8%D1%8F%2C%20%D0%BA%D0%BB%D0%B0%D1%81%D1%82%D0%B5%D1%80%D0%B8%D0%B7%D0%B0%D1%86%D0%B8%D1%8F.md#ml-как-оптимизационная-задача>) сводится к $\hat\theta = \arg\min_\theta \hat R(\theta) + \lambda\Omega(\theta)$, то есть к минимизации лосса с регуляризацией.

## Развёрнуто

### Обучение с учителем
Дано: обучающая выборка $\{(x_i, y_i)\}_{i=1}^n$, $x_i\in\mathcal X$, $y_i\in\mathcal Y$, предполагаемые независимые наблюдения из неизвестного распределения $P(x, y)$.
Цель — найти модель $f: \mathcal X\to\mathcal Y$, обобщающуюся на новые данные (минимизирующую обобщённый риск $\mathbb E_{(x,y)}L(f(x), y)$).

**Классификация**: $\mathcal Y$ — конечно ($\{0, 1\}, \{1, \dots, K\}$).
- Бинарная (спам/не спам, болен/здоров).
- Мультикласс (распознавание цифр, тематика текста).
- Мультилейбл — несколько меток одновременно.

Лоссы: 0-1 (теоретически), кросс-энтропия, hinge.

Примеры моделей: логистическая регрессия, SVM, деревья решений, случайный лес, бустинг (XGBoost, LightGBM), нейронные сети.

**Регрессия**: $\mathcal Y\subseteq\mathbb R$ — непрерывное.
- Линейная регрессия.
- Полиномиальная.
- KNN-регрессия.
- Ridge/Lasso.
- Деревья и бустинг.
- Нейронные сети.

Лоссы: MSE (квадратичная), MAE (абсолютная), Huber (компромиссная).

### Обучение без учителя
Только $\{x_i\}$, без меток.

**Кластеризация** — разбить точки на группы похожих:
- k-means;
- иерархическая кластеризация (агломеративная/дивизивная);
- DBSCAN, OPTICS (по плотности);
- спектральная кластеризация;
- Gaussian Mixture Models (вероятностная).

**Понижение размерности**:
- PCA (линейное);
- t-SNE, UMAP (нелинейное, для визуализации);
- автокодировщики.

**Оценка плотности** (density estimation):
- параметрическая (подгонка под распределение);
- ядерная оценка плотности (KDE);
- normalizing flows.

**Поиск аномалий** — отдельные «выбросы» по сравнению с большинством.

**Поиск ассоциаций** (association rules) — Apriori, FP-growth.

### Другие парадигмы
- **Обучение с подкреплением (RL)**: агент → среда, награды → политика. Q-learning, policy gradient, актор-критик.
- **Полу-супервизированное** (semi-supervised): мало размеченных + много неразмеченных.
- **Самосупервизированное** (self-supervised): создаём «искусственные» метки из самих данных (предсказание соседнего слова, маскированный токен).
- **Активное** обучение (active learning) — модель выбирает, какие точки ей разметить.
- **Трансферное** обучение, meta-learning, few-shot.

### ML как оптимизационная задача
Формализация: пространство гипотез $\mathcal F = \{f_\theta\}$, параметры $\theta\in\Theta$. **Эмпирический риск**:
$$\hat R(\theta) = \frac{1}{n}\sum_{i=1}^n L(f_\theta(x_i), y_i).$$
Обучение — минимизация $\hat R$:
$$\hat\theta = \arg\min_\theta \hat R(\theta) + \lambda\Omega(\theta).$$
$\Omega$ — регуляризация (L1/Lasso, L2/Ridge, dropout как стохастическая регуляризация).

Цель — приближение к минимуму **истинного** риска $R(\theta) = \mathbb E_{(x,y)}L(f_\theta(x),y)$. Разница $|R - \hat R|$ ограничивается через теорию Вапника-Червоненкиса (VC dimension), Радемахеровские сложности и т.д.

### Решение оптимизационной задачи
- **Аналитическое**: линейная регрессия — $\hat\theta = (X^\top X)^{-1}X^\top Y$, если $X^\top X$ невырождена; иначе используют псевдообратную матрицу или регуляризацию (ridge).
- **Градиентные методы**: SGD, momentum, Adam, RMSProp.
- **Специализированные**: LBFGS, мажоризация–минимизация, EM, координатный спуск (Lasso).
- **Деревья и бустинг** — жадные алгоритмы.

Стохастический градиент: на каждой итерации используется случайная мини-партия (mini-batch) для оценки градиента — позволяет масштабировать на большие выборки.

### Связь компонентов
- **Лосс** $L$ — что мы хотим минимизировать.
- **Гипотеза** $\mathcal F$ — какие функции рассматриваем.
- **Регуляризация** $\Omega$ — контроль сложности (борьба с переобучением).
- **Алгоритм оптимизации** — как находим $\hat\theta$.
- **Метрика** на тесте — что измеряем (может отличаться от лосса!).

### Пример
Задача: предсказать цену недвижимости по площади и району.
- Тип: регрессия (цена непрерывна).
- Лосс: MSE.
- Гипотеза: линейная регрессия $f_\theta(x) = \theta^\top x$.
- Регуляризация: Ridge $\Omega(\theta) = \|\theta\|_2^2$.
- Оптимизация: closed-form или SGD.
- Метрика: RMSE на тестовой выборке.

### Подводные камни
- **Утечка данных** (data leakage): признаки, недоступные на момент прогноза, или просочившиеся метки.
- Распределение в проде ≠ обучение → деградация (concept drift).
- Лосс ≠ метрика: оптимизируем кросс-энтропию, а оценивают по AUC — нужно следить.
- Обучение без учителя — нет «правильного» ответа, валидация субъективна.
