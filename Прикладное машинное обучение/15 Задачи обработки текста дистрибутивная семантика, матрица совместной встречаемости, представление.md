# Задачи обработки текста: дистрибутивная семантика, матрица совместной встречаемости, представление слов в виде векторов малой размерности. Модель Word2vec.

## TL;DR
- **Задачи обработки текста:** текст переводят в числовые признаки для поиска, классификации, рекомендаций и других NLP-задач; типовые применения перечислены в <a href="#применения">применениях</a>.
- **Дистрибутивная семантика:** <a href="#дистрибутивная-семантика">значение слова приближается его контекстами</a>, поэтому похожие контексты дают близкие векторы.
- **Матрица совместной встречаемости:** <a href="#матрица-совместной-встречаемости">матрица со-встречаемости</a> хранит частоты совместного появления слов и может нормироваться через PMI/PPMI.
- **Представление слов в виде векторов малой размерности:** разреженную статистику сжимают в плотные эмбеддинги, например через <a href="#понижение-размерности-lsa">LSA</a> или нейросетевые модели.
- **Модель Word2vec:** <a href="#word2vec-mikolov-2013">Word2vec</a> учит CBOW или Skip-Gram и ускоряет обучение negative sampling либо hierarchical softmax.

## Развёрнуто

<a id="дистрибутивная-семантика"></a>

### Дистрибутивная семантика
**Дистрибутивная гипотеза (Харрис, 1954):** "Слова с похожими контекстами имеют похожие значения." Это операциональный способ построить векторные представления слов через статистику корпуса, без ручной семантической разметки.

<a id="матрица-совместной-встречаемости"></a>

### Матрица совместной встречаемости

$M_{ij}$ = сколько раз слова $i$ и $j$ встретились в окне $\pm k$. Симметричная, $V \times V$, разреженная.

**PMI (Pointwise Mutual Information):** $$\text{PMI}(i, j) = \log \frac{P(i, j)}{P(i) P(j)}.$$

Нормирует частоты, выделяет реально связанные пары. $> 0$ — связь, $\approx 0$ — независимы.

**PPMI** = $\max(\text{PMI}, 0)$. Отрицательные значения обрезаются (шумные, плюс $\log 0 = -\infty$ ломает).

<a id="понижение-размерности-lsa"></a>

### Понижение размерности (LSA)

Матрица $V \times V$ огромная и разреженная → применяем **SVD**: $M \approx U \Sigma V^\top$. Берём топ-$d$ сингулярных значений → плотные эмбеддинги в $\mathbb{R}^{d}$ ($d = 50$-$300$). Это классический подход до Word2vec.

<a id="word2vec-mikolov-2013"></a>

### Word2vec (Mikolov, 2013)

Нейросетевой подход. Две архитектуры:

- **CBOW** — по контексту предсказать центральное слово.
- **Skip-Gram** — по слову предсказать контекст. Обычно лучше работает.

Полный softmax по словарю $|V| \approx 10^5$ слишком дорогой. Ускорения:

**Negative Sampling (главное на практике):** $$\log\sigma(u_o^\top v_c) + \sum_{i=1}^k \log\sigma(-u_{w_i}^\top v_c).$$

Учим различать позитивные пары (центр + реальный контекст) от негативных (центр + случайные слова из шума $\propto f(w)^{3/4}$). $k = 5$-$20$ негативов вместо суммы по всему словарю. Приближение, но быстрое.

**Hierarchical Softmax:** словарь как бинарное дерево (Хаффмана), каждое слово — лист. $P(w)$ = произведение сигмоид по пути от корня до листа. Сложность $O(\log V)$ вместо $O(V)$. Точное распределение, но на практике вытеснен negative sampling.

### Свойства Word2vec

- Плотные векторы $\mathbb{R}^{300}$.
- **Косинусная близость** отражает семантическую похожесть.
- **Линейные регулярности:** $v_{\text{king}} - v_{\text{man}} + v_{\text{woman}} \approx v_{\text{queen}}$. Векторы кодируют семантические "оси" (гендер, число, страны-столицы).
- Кластеризация по темам.

### GloVe (Pennington, 2014)

Гибрид LSA и Word2vec: использует **полную матрицу со-встречаемости** + учит **плотные векторы с линейными свойствами**.

Цель: $v_i^\top u_j \approx \log M_{ij}$. Геометрическая близость векторов соответствует статистической связи слов.

Loss: $$J = \sum_{i,j} f(M_{ij}) (v_i^\top u_j + b_i + b_j - \log M_{ij})^2.$$

$f(M_{ij})$ гасит вес очень редких (шум) и очень частых пар. Логарифм используется, чтобы разности векторов кодировали отношения частот — отсюда линейные регулярности.

### Связь Word2vec и GloVe

Levy & Goldberg (2014): Skip-Gram с Negative Sampling неявно факторизует **сдвинутую PPMI-матрицу**: $$u_w^\top v_c \approx \text{PMI}(w, c) - \log k.$$

GloVe явно факторизует $\log M_{ij}$. То есть **оба подхода под капотом делают одно** — учат факторизации PMI-подобных матриц. Просто разными методами: Word2vec через стохастический градиентный подход на локальных окнах, GloVe через MSE на полной матрице.

### Расширения

- **FastText** (Bojanowski, 2017): эмбеддинг слова = сумма эмбеддингов символьных n-грамм. Решает OOV (out-of-vocabulary), работает на морфологически богатых языках.
- **Контекстные эмбеддинги** (ELMo, BERT, GPT): представление слова **зависит от контекста**, а не статично. Решает многозначность ("bank" — банк или берег). Заменили статические Word2vec/GloVe в большинстве NLP-задач.

<a id="применения"></a>

### Применения

- Семантический поиск, similarity.
- Признаки для классификации (sentiment, topic).
- Машинный перевод (входной слой).
- Рекомендации (item2vec, doc2vec).
- Биоинформатика (последовательности белков).

### Подводные камни

- Word2vec/GloVe не учитывают многозначность (одно слово = один вектор).
- Эмбеддинги наследуют **bias** корпуса (гендерные/расовые).
- Косинусная близость = "дистрибутивная близость", не лингвистическая синонимия — антонимы тоже близки.
- OOV: незнакомые слова — нет вектора. Решается FastText или BPE-токенизацией.

### Главное на экзамене

1. **Дистрибутивная гипотеза Харриса** — одна фраза.
2. **Матрица со-встречаемости + PMI/PPMI** — формула и зачем.
3. **Word2vec:** CBOW vs Skip-Gram, негативное сэмплирование зачем.
4. **Линейные регулярности** как ключевое свойство.
5. **GloVe:** факторизация $\log M_{ij}$ через MSE.
6. **Связь** Word2vec и GloVe через PMI.
