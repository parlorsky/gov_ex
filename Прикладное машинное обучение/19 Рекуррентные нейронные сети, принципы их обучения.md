# Рекуррентные нейронные сети, принципы их обучения. Сложности применения рекуррентных нейронных сетей. Модуль LSTM.

## TL;DR
**Рекуррентные нейронные сети (RNN)** — для последовательностей переменной длины. На каждом шаге обновляют скрытое состояние: $h_t = \sigma(W_h h_{t-1} + W_x x_t + b)$. Обучаются **алгоритмом BPTT** (backprop через время) — разворачиванием по шагам и стандартным backprop. Главные проблемы: **исчезающий и взрывающийся градиент**, ограниченная память. **LSTM** добавляет состояние ячейки $c_t$ и три гейта (input, forget, output): $c_t = f_t\odot c_{t-1} + i_t\odot \tilde c_t$, $h_t = o_t\odot\tanh c_t$. Гейты позволяют сети «помнить» долго и контролировать поток информации. **GRU** — упрощённая версия с двумя гейтами. Современная альтернатива — Transformers с self-attention.

## Развёрнуто

### Рекуррентная сеть (Vanilla RNN)
Вход — последовательность $x_1, x_2, \dots, x_T$. Скрытое состояние $h_t\in\mathbb R^H$:
$$h_t = \tanh(W_h h_{t-1} + W_x x_t + b),$$
$$y_t = W_y h_t + c.$$
$h_0 = 0$ или обучается. Параметры $\theta = (W_h, W_x, W_y, b, c)$ **разделяются между шагами** — отсюда «рекуррентная».

Применение:
- классификация последовательности (один выход в конце);
- последовательность в последовательность (seq2seq, например, машинный перевод — encoder + decoder);
- последовательность в метку на каждом шаге (POS-tagging).

### Обучение: BPTT (Backpropagation Through Time)
Сеть «разворачивается» по времени в глубокую сеть на $T$ шагов с разделёнными весами. Backprop вычисляет градиент стандартно. Градиент по $W_h$:
$$\frac{\partial L}{\partial W_h} = \sum_t \frac{\partial L_t}{\partial h_t}\cdot \prod_{s>t}\frac{\partial h_s}{\partial h_{s-1}}\cdot \frac{\partial h_t}{\partial W_h}.$$

**Truncated BPTT**: для очень длинных последовательностей разворачиваем только на $k$ шагов назад — компромисс между точностью и памятью.

### Сложности применения

**1. Vanishing gradient (исчезающий)**: $\partial h_s/\partial h_{s-1} = W_h^\top \mathrm{diag}(\sigma'(\cdot))$. При $\|W_h\| < 1$ произведение $\prod$ экспоненциально стремится к 0 — сеть не учится зависимостям на больших временных промежутках.

**2. Exploding gradient (взрывающийся)**: при $\|W_h\| > 1$ произведение растёт экспоненциально. Решение — **gradient clipping**: $g\to g\cdot\min(1, \tau/\|g\|)$.

**3. Ограниченная память**: vanilla RNN запоминает примерно 10–20 шагов на практике; для больших контекстов нужны LSTM/GRU/Transformer.

**4. Последовательность вычислений**: невозможно параллелить шаги по $t$ — медленно на GPU.

**5. Long-term dependencies**: трудно обучить связи между удалёнными шагами без специальной архитектуры.

### LSTM (Long Short-Term Memory, Hochreiter & Schmidhuber, 1997)
Добавляет **клеточное состояние** $c_t$ — «магистраль» с минимальной модификацией, плюс три гейта.

**Гейты** (sigmoid → значения в $[0, 1]$):
- **forget** $f_t = \sigma(W_f[h_{t-1}, x_t] + b_f)$ — что забыть из $c_{t-1}$;
- **input** $i_t = \sigma(W_i[h_{t-1}, x_t] + b_i)$ — что записать;
- **output** $o_t = \sigma(W_o[h_{t-1}, x_t] + b_o)$ — что показать наружу.

**Кандидат на запись**: $\tilde c_t = \tanh(W_c[h_{t-1}, x_t] + b_c)$.

**Обновление состояний**:
$$c_t = f_t\odot c_{t-1} + i_t\odot \tilde c_t,$$
$$h_t = o_t\odot \tanh(c_t).$$

$\odot$ — поэлементное умножение. $[h_{t-1}, x_t]$ — конкатенация.

**Зачем работает**: путь $c_{t-1}\to c_t$ — **аддитивный** с гейтом $f_t$, поэтому градиент $\partial c_t/\partial c_{t-1}\approx f_t$ — не затухает экспоненциально, если $f_t\approx 1$. Это и решает проблему длинных зависимостей.

### GRU (Gated Recurrent Unit, Cho et al., 2014)
Упрощённый LSTM с **двумя** гейтами:
- **reset** $r_t = \sigma(W_r[h_{t-1}, x_t])$;
- **update** $z_t = \sigma(W_z[h_{t-1}, x_t])$.

$$\tilde h_t = \tanh(W[r_t\odot h_{t-1}, x_t]),\quad h_t = (1-z_t)\odot h_{t-1} + z_t\odot \tilde h_t.$$
Меньше параметров, в среднем сравним с LSTM.

### Двунаправленные RNN (BiLSTM/BiGRU)
Два прохода: вперёд и назад. На каждом шаге $h_t = [h_t^{\to}, h_t^{\leftarrow}]$. Используются для задач, где будущий контекст доступен (POS, NER, классификация документа).

### Encoder-decoder (seq2seq) и attention
Encoder сжимает входную последовательность в вектор $z$, decoder генерирует выход по $z$. Слабость — bottleneck в $z$ при длинных входах.

**Attention** (Bahdanau, Luong) — на каждом шаге декодера вычисляется веса по всем состояниям энкодера, контекст — взвешенная сумма. Стало основой для **Transformer**, полностью заменившего RNN в большинстве задач NLP.

### Применение RNN
Раньше — стандарт для NLP (LM, перевод, тегирование), временных рядов, генерации текста. Сейчас в NLP их вытеснили Transformer, но RNN/LSTM/GRU остаются полезными:
- временные ряды с малыми ресурсами;
- онлайновая обработка потоков;
- небольшие модели на устройствах.

### Пример (LSTM для классификации текста)
```python
import torch.nn as nn
class TextLSTM(nn.Module):
    def __init__(self, vocab, emb=128, hid=256, classes=2):
        super().__init__()
        self.emb = nn.Embedding(vocab, emb)
        self.lstm = nn.LSTM(emb, hid, batch_first=True, bidirectional=True)
        self.fc = nn.Linear(2*hid, classes)
    def forward(self, x):
        e = self.emb(x)
        out, (h, c) = self.lstm(e)
        last = out.mean(dim=1)  # GlobalAvgPool по времени
        return self.fc(last)
```

### Подводные камни
- Без gradient clipping LSTM также может «взорваться» на длинных последовательностях.
- Инициализация forget gate bias положительной величиной (1 или 2) ускоряет обучение.
- BiLSTM нельзя использовать в задачах online или с маскированным будущим (генерация).
- При длине $T$ большой обучение остаётся медленным — Transformer быстрее за счёт распараллеливания.
