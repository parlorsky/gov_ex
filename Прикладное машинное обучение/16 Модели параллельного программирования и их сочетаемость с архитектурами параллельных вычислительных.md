## Развёрнуто

### Архитектуры

**По памяти:**
- **Shared memory** (общая RAM): SMP, многоядерные CPU. Все потоки видят одни данные напрямую.
- **Distributed memory** (своя память + сеть): кластеры. Обмен через сетевые сообщения.

**Особый случай:**
- **GPU (SIMD/SIMT)**: тысячи лёгких потоков, одна инструкция на разных данных.

### Классификация Флинна
- **SISD** — обычный CPU (один поток).
- **SIMD** — одна инструкция, много данных (GPU, AVX).
- **MIMD** — много инструкций, много данных (многоядерные CPU, кластеры).
- **MISD** — редкая теоретическая категория.

### Модели программирования и соответствие архитектурам

|Архитектура|Модель|
|---|---|
|Shared memory (SMP)|OpenMP, std::thread, pthreads|
|Distributed memory (кластер)|MPI (message passing)|
|GPU|CUDA, OpenCL|
|Big data на кластере|MapReduce, Spark, Dask|

**Гибриды:** MPI + OpenMP (кластер с многоядерными узлами), MPI + CUDA (распределённое DL — PyTorch DDP, Horovod).

### Декомпозиция задач
- **Data parallel** — одна операция на разных частях данных (батчи в ML, обработка пикселей). Подходит SIMD/GPU.
- **Task parallel** — разные операции параллельно, независимые задачи (потоки сервера, игровой движок).
- **Pipeline** — стадии конвейера, данные текут через них (CPU инструкции, ETL, видеообработка). Частный случай task parallel с жёсткой последовательностью.
- **Master/worker** — один раздаёт задания, остальные считают. Для нерегулярных задач, динамической балансировки.
- **Domain decomposition** — расчётная область делится геометрически, обмен по границам. Стандарт для PDE-симуляций (CFD, погода).

### Закон Амдаля
$$S(p) = \frac{1}{(1-f) + f/p}.$$

$f$ — параллелизуемая доля. При $p \to \infty$ → $S \to 1/(1-f)$ — **жёсткий потолок** из-за непараллелизуемой части.

Пример: $f = 0.95$ → максимум $S = 20$ даже на бесконечности процессоров. Высокая $f$ критична.

### Закон Густафсона
$$S(p) = (1-f) + p \cdot f.$$

**Слабое масштабирование** — задача растёт с числом процессоров. Ускорение **линейное** по $p$. Окупается, когда задача может масштабироваться (детализация сетки, больше данных).

### Метрики
- **Speedup** $S(p) = T_1 / T_p$.
- **Efficiency** $E(p) = S(p)/p$.
- **Strong scaling** — фикс. задача, растёт $p$.
- **Weak scaling** — задача растёт с $p$.

### Подводные камни
- **Race conditions** — самые тяжёлые баги в shared memory.
- **Network latency** в distributed memory — может убить весь параллелизм, если коммуникации много.
- **False sharing** — кеш-эффекты при близких адресах.
- **Balance** — плохая балансировка нагрузки убивает эффективность.
- **Granularity** — слишком мелкие задачи → overhead больше работы.
