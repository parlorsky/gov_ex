## Развёрнуто

### Постановка кластеризации
Дано $\{x_i\}_{i=1}^n\subseteq\mathbb R^d$, без меток. Найти разбиение $\{C_1,\dots,C_K\}$ так, чтобы внутри $C_k$ объекты были «похожи», между — «непохожи». Похожесть — метрика (евклидова, косинусная) или кернел.

### Алгоритмы кластеризации

**k-means**. Минимизация суммы квадратов расстояний:
$$J = \sum_{k=1}^K\sum_{x\in C_k}\|x - c_k\|^2.$$
Алгоритм Ллойда:
1. Инициализировать центры $c_1,\dots,c_K$.
2. Назначение: $C_k = \{x: \|x - c_k\|\le \|x - c_j\|\,\forall j\}$.
3. Обновление: $c_k = \tfrac{1}{|C_k|}\sum_{x\in C_k} x$.
4. Повторять до сходимости.

Сложность $O(nKd\cdot \text{iter})$. **k-means++** — улучшенная инициализация, более устойчивая. Чувствителен к выбросам и формам кластеров (любит сферические).

**Иерархическая кластеризация**: агломеративная (снизу вверх) или дивизивная. Метрики связи: single, complete, average, Ward. Результат — дендрограмма; число кластеров выбирается разрезом.

**DBSCAN** (Density-Based): core-точка имеет в $\varepsilon$-окрестности не меньше minPts соседей; кластер образуется из core-точек, связанных по плотности, и достижимых от них граничных точек. Не требует $K$, находит произвольные формы, выделяет шум. Параметры: $\varepsilon$, minPts.

**GMM (Gaussian Mixture Model)**: $p(x) = \sum \pi_k\mathcal N(x|\mu_k, \Sigma_k)$, обучается EM-алгоритмом. Soft-кластеризация (вероятности принадлежности).

**Spectral clustering**: собственные векторы лапласиана графа сходства; затем k-means в спектральном пространстве. Хорошо для нерегулярных кластеров.

**Прочие**: Mean-shift, OPTICS, BIRCH, HDBSCAN, аффинное распространение.

### Подбор числа кластеров
**Метод локтя (elbow)**: график $J(K)$ как функция $K$; ищем «излом». Субъективно.

**Силуэт (silhouette)**: для каждого $x_i$
$$s_i = \frac{b_i - a_i}{\max(a_i, b_i)},$$
$a_i$ — среднее расстояние до точек своего кластера, $b_i$ — минимум по другим кластерам от среднего расстояния до точек этого чужого кластера. $s\in[-1, 1]$, ближе к 1 — лучше. Усреднённый по всем — глобальная метрика.

**Gap statistic**: сравнение $J(K)$ с тем же на случайных равномерных данных.

**Calinski-Harabasz**: отношение межкластерной дисперсии к внутрикластерной.

**Davies-Bouldin**: средняя «разделимость» соседних кластеров.

**Bayesian Information Criterion (BIC)** для GMM.

### Метрический классификатор kNN
Для классификации $x$:
1. Найти $k$ ближайших обучающих примеров (по выбранной метрике).
2. Метку взять как мажоритарную (или вероятности по долям).

Для регрессии — среднее $y$ соседей. Модель **lazy**: обучение тривиально, всё в предсказании.

**Метрики**: евклидова, манхэттенская, Минковского, Махаланобиса, косинусная. Должна быть выбрана с учётом природы данных и масштабирована (StandardScaler!).

### Подбор числа $k$
- Малое $k$ — высокая дисперсия (чувствительность к шуму).
- Большое $k$ — высокий bias, сглаживание.
- **Кросс-валидация**: перебираем $k = 1, 3, 5, \dots$ и выбираем по метрике (accuracy, F1).
- Эмпирическое правило: $k\approx\sqrt n$ как стартовая точка.
- Чётное $k$ может приводить к ничьим — лучше нечётное (для бинарной).

### Обобщения kNN
- **Weighted kNN**: голос соседа взвешивается обратно расстоянию (`weights='distance'`).
- **Radius Neighbors**: вместо $k$ берутся все соседи в радиусе $r$.
- **Locally Weighted Regression**: KNN-регрессия с локально подобранной моделью.
- **NearestCentroid**: класс ассоциируется с центроидом своих точек (упрощённый kNN).
- **KNN с предобработкой**: PCA/UMAP для понижения размерности перед поиском.

### Ускорение поиска соседей
- **KD-Tree**: бинарное дерево разбиений по осям; средняя сложность $O(\log n)$ на запрос при $d$ малом.
- **Ball-Tree**: гиперсферические разбиения; лучше при больших $d$.
- **HNSW, Annoy, FAISS** — приближённые методы для $d$ большого / больших $n$.

В Scikit-Learn — параметр `algorithm` (`auto`, `ball_tree`, `kd_tree`, `brute`).

### Методы метрической классификации в Scikit-Learn
- `KNeighborsClassifier`, `KNeighborsRegressor` — стандартный kNN.
- `RadiusNeighborsClassifier/Regressor` — радиусный.
- `NearestCentroid` — классификация по ближайшему центроиду.
- `KNeighborsTransformer` — построение разрежённой матрицы соседей для других моделей.
- `NeighborhoodComponentsAnalysis (NCA)` — обучаемая метрика, увеличивающая kNN-точность.

Кластеризация:
- `KMeans`, `MiniBatchKMeans`.
- `AgglomerativeClustering` (Ward, complete, average, single).
- `DBSCAN`, `OPTICS`, `HDBSCAN` (extra).
- `GaussianMixture`.
- `SpectralClustering`.

Метрики кластеризации в `sklearn.metrics`:
- неконтролируемые: `silhouette_score`, `calinski_harabasz_score`, `davies_bouldin_score`;
- при наличии истинной разметки: `adjusted_rand_score`, `adjusted_mutual_info_score`, `normalized_mutual_info_score`, `homogeneity/completeness`.

### Пример: kNN на Iris
```python
from sklearn.datasets import load_iris
from sklearn.model_selection import cross_val_score
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

X, y = load_iris(return_X_y=True)
pipe = Pipeline([("sc", StandardScaler()), ("knn", KNeighborsClassifier())])
for k in [1, 3, 5, 7, 11]:
    pipe.set_params(knn__n_neighbors=k)
    print(k, cross_val_score(pipe, X, y, cv=5).mean())
```

### Подводные камни
- kNN страдает от **проклятия размерности**: при $d$ большом расстояния «выравниваются». Перед kNN — PCA/feature selection.
- Не масштабированные признаки ломают метрику.
- Для больших $n$ kNN медленный — нужны индексные структуры или приближённые методы.
- В кластеризации нет правильного ответа — выбор алгоритма и метрики качества субъективен.
- k-means чувствителен к инициализации — стоит запускать многократно (`n_init` в Scikit-Learn).
